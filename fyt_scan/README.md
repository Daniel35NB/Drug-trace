# 丰源堂药品追溯扫码系统

一款基于 uni-app 开发的 Android 药品追溯码扫码上传应用，专为零售药店设计，用于扫描药品追溯码并上传至"码上放心"平台（淘宝开放平台 API）。

## 📋 功能特性

### 核心功能
- ✅ **扫码上传**：调用原生摄像头扫描 20 位药品追溯码
- ✅ **自动识别**：智能校验追溯码格式
- ✅ **连续扫码**：上传成功后自动继续下一次扫码
- ✅ **离线缓存**：网络异常时自动缓存，恢复后同步
- ✅ **重试机制**：指数退避重试（1s → 2s → 4s）
- ✅ **数据统计**：今日扫码数、失败数、总成功数实时统计

### 用户体验
- ✅ **多感官反馈**：扫码音效 + 振动提示
- ✅ **状态可视化**：清晰的扫码状态指示
- ✅ **历史记录**：最近 30 条扫码记录可查
- ✅ **日志系统**：完整的操作日志和错误追踪
- ✅ **自定义导航栏**：适配各种屏幕状态栏

### 安全可靠
- ✅ **配置隔离**：敏感信息独立配置文件
- ✅ **签名加密**：严格遵循淘宝 API 签名规范
- ✅ **数据持久化**：防止应用崩溃导致数据丢失
- ✅ **网络检测**：实时监测网络状态变化

## 🛠 技术栈

- **框架**：uni-app (Vue 3)
- **开发工具**：HBuilderX 5.03+
- **目标平台**：Android（兼容鸿蒙）
- **API**：淘宝开放平台 - 码上放心追溯系统

## 🚀 快速开始

### 1. 环境准备

- 安装 [HBuilderX](https://www.dcloud.io/hbuilderx.html) 5.03+
- 准备 Android 测试设备或模拟器

### 2. 克隆项目

```bash
git clone <repository-url>
cd fyt_scan
```

### 3. 配置密钥

```bash
# 复制配置示例
cp config.example.js config.js

# 编辑 config.js，填入真实的 appSecret
```

**⚠️ 重要安全提示：**
- `config.js` 包含敏感信息，已被加入 `.gitignore`
- 切勿将 `config.js` 上传到公共代码仓库
- 生产环境建议使用环境变量管理密钥

### 4. 运行项目

1. 用 HBuilderX 打开项目目录
2. 连接 Android 设备或启动模拟器
3. 点击菜单 `运行` → `运行到手机或模拟器`
4. 选择已连接的设备

### 5. 打包发布

1. 在 HBuilderX 中点击 `发行` → `原生 App-云打包`
2. 选择 Android 平台
3. 配置包名：`com.fengyuantang.retail`
4. 上传证书（可使用公共测试证书或自有证书）
5. 点击打包，下载 APK 文件

## 📱 使用说明

### 扫码流程

1. 打开应用，点击 **"📷 扫码销售"** 按钮
2. 对准药品追溯码（20 位数字条形码）
3. 自动识别并上传
4. 听到提示音且显示"✓"表示成功

### 离线模式

- 当网络断开时，扫码数据会自动缓存到本地
- 网络恢复后，系统会每 30 秒自动尝试同步
- 也可手动点击 **"🔁 重试失败"** 按钮立即同步

### 查看日志

1. 点击右上角 **"📋 日志"** 按钮
2. 查看系统运行日志和错误信息
3. 支持复制到剪贴板导出

## 🔧 调试功能

应用内置调试按钮组（位于主界面中部）：

- **测试 API**：测试与淘宝 API 的连接状态
- **清日志**：清空所有系统日志

## 📝 常见问题

### Q: 扫码没反应？
A: 检查相机权限是否开启，确保光线充足，追溯码清晰无损。

### Q: 上传失败怎么办？
A: 
1. 检查网络连接
2. 确认 config.js 配置正确
3. 查看日志中的错误信息
4. 等待自动重试或手动点击"重试失败"

### Q: 如何关闭连续扫码？
A: 在 `config.js` 中将 `autoContinue` 设置为 `false`。

### Q: 支持鸿蒙系统吗？
A: 支持。uni-app 编译的 Android 应用可在鸿蒙系统上正常运行。
